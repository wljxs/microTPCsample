# 模拟数据与分析逻辑核对说明

## 1. 核对范围

本说明核对以下链路：

```text
TrackHeed/人工轨迹
  -> 初级电子位置 tree3
  -> 漂移到达位置与抽样库波形
  -> 波形卷积并写入 tree.waveforms_conv
  -> simulationanalysis.C 读取波形
  -> 15 点采样、解卷积、按条聚类
  -> microTPC 位置/斜率与真值比较
```

主要文件：

- `simulation/microTPCsimulation.C`
- `analysis/src/simulationanalysis.C`
- `simulation/analysis/matix.C`（通过 `matrix.root` 提供解卷积模板）

## 2. 模拟逻辑概述

模拟程序使用 5 mm 气隙、20 条 x 条和 20 条 y 条，条间距设为 `0.04`。每个事件固定从 `x0 = 2*pitch`、`y0 = 2*pitch`、`z0 = 0.49` 开始，轨迹方向为：

```cpp
dx = 0;
dy = sin(20 degree);
dz = -cos(20 degree);
```

因此实际模拟轨迹是在 **y-z 平面** 内倾斜，x 方向没有倾角。真值斜率定义为 `x = kx*z + bx`、`y = ky*z + by`，其中 `kx = 0`、`ky = -tan(20 degree)`。

每个事件人为产生 1000 个初始电子，`ze0` 从约 `0.05` 到 `4.95` 均匀分布，每隔 `0.1` 分布一层。电子随后漂移，并根据漂移终点从抽样库取波形，累加到相邻条上。最终保存：

- `tree`：条号、方向、原始波形、卷积波形、真值 `k/b`；
- `tree2`：漂移终点；
- `tree3`：初始电离位置。

## 3. 分析逻辑概述

`simulationanalysis.C` 当前执行以下步骤：

1. 从 `simulation/analysis/matrix.root` 读取 5 个 tier 的独立模板；
2. 将模板每隔 5 个点抽取一次，构造 x/y 解卷积矩阵；
3. 从模拟文件读取 `tree.waveforms_conv`；
4. y 方向先乘 `-1`，再抽取每隔 5 个点的数据并乘以 `1000`；
5. 用 NNLS/Lasso 解出 5 个 tier 的电荷；
6. 将相邻条聚成 cluster；
7. 用 `cluster.pos` 和 `cluster.microTPCposition` 与模拟保存的 `k/b` 比较；
8. 从 `tree3` 统计每层初始电子数和平均位置。

## 4. 已确认的问题

### 4.1 `tree3` 的坐标被错误地乘以 10，导致层统计基本失真

模拟中 COMSOL 使用的单位是 mm，`pitch = 0.04`、`z0 = 0.49`，初始电子的 `ze0` 约为 `0.05--4.95`，本身已经是 mm。

但分析中使用：

```cpp
double z = (*ze0)[i] * 10;
sumX[layer] += (*xe0)[i] * 10;
sumY[layer] += (*ye0)[i] * 10;
```

这会把 z 范围变成约 `0.5--49.5 mm`，而分层范围仍是 `0--5 mm`。结果是绝大多数电子被强行放进最后一层，2D 图中的 z 也会溢出到直方图上界之外。

**结论：这是确定的严重错误。** 如果模拟坐标确实保持 mm，应删除这些 `*10`。x/y 也应删除 `*10`，并重新确认图的横坐标范围。

### 4.2 采样点数量少了一个，14 点与 15 点配置不一致

分析设置：

```cpp
const int n_sig_samples = 15;
```

但抽样条件为：

```cpp
if (j % 5 == 0 && j < n_sig_samples * 5 - 1)
```

对于 `n_sig_samples = 15`，实际抽到的下标是 `0, 5, ..., 65`，共 14 个点，而不是 15 个点。模板构造处也使用了同样的条件。

这会使输入波形长度和解卷积模板的预期长度不一致，可能导致矩阵维度不匹配、最后一个采样点被丢弃或结果发生隐式截断。

**建议：** 将条件改为：

```cpp
if (j % 5 == 0 && j < n_sig_samples * 5)
```

模板抽样处同样修改，并在进入解卷积前显式检查 `waveform.size() == n_sig_samples`。

### 4.3 模拟卷积循环存在首点/末点边界偏移

模拟卷积为：

```cpp
for (int j = 0; j < k; ++j) {
    int idx = k - j;
    convx += vec_sigx[indexx][idx] * kernel_vec[j];
}
```

当 `k = 0` 时没有任何贡献；当 `k > 0` 时使用了 `signal[k]` 到 `signal[1]`，没有使用通常因果卷积中的 `signal[k-j]` 在 `j=0` 时的当前点与 `j=k` 时的首点组合。是否需要包含 `j=k` 取决于定义，但当前实现明确存在一个采样边界偏移，而且没有注释说明这是有意的延迟。

**建议：** 明确卷积定义。若要使用标准离散因果卷积，应考虑：

```cpp
for (int j = 0; j <= k; ++j) {
    int idx = k - j;
    convx += vec_sigx[indexx][idx] * kernel_vec[j];
}
```

同时让模板生成程序使用完全相同的卷积定义，否则解卷积矩阵与模拟波形不匹配。

### 4.4 模拟波形与解卷积模板的归一化关系没有被严格验证

模拟输出卷积波形时乘了：

```cpp
vec_sigx_conv[indexx][k] = convx * tstep;
```

分析读取模板时又乘了：

```cpp
sig_temp[0].push_back((*sigx_conv)[i] * scale);
```

分析波形还乘了 `1000`。这些缩放因子未在同一处集中定义，也没有自动检查模板与输入波形的幅度比例。

如果 `matrix.root` 来自不同的卷积核、时间步长或单位，NNLS 求出的 tier 电荷会带有不可控的整体比例，甚至影响 `rnorm/charge`。

**建议：** 在模拟文件和矩阵文件中记录 `tstep`、卷积核、采样间隔、幅度单位和整体 scale；分析启动时检查这些元数据，而不是依靠 `4200`、`1000` 这样的硬编码。

### 4.5 真值残差公式混用了单位和经验比例，当前不能认为是严格真值比较

分析中使用了多种换算：

```cpp
cluster.pos[tier] * 0.4 + 0.2
b[type] * 10
k[type] * thickness / n_tier * (tier + 0.5)
```

同时绘图预测又使用：

```cpp
predb = cluster.b + 0.2 * 2.5;
hit = k[type] * z * 2.5 + b[type] * 25;
```

这些公式彼此并不使用同一坐标定义。尤其是模拟的 `b` 是由 mm 坐标直接计算的，而残差中使用 `b*10`；绘图中又出现 `b*25`。因此图中蓝色真值轨迹、残差直方图和微 TPC 残差可能对应不同的坐标系。

**建议：** 统一定义：

```text
真实坐标：x_true(z) = k_true * z + b_true，单位 mm
重建坐标：x_reco = (strip_position + strip_offset) * pitch_mm
残差：residual = x_reco - x_true(z)
```

所有方向都使用同一公式，删除没有物理定义的 `*10`、`*25`、`*2.5`，除非这些因子明确代表坐标索引到 mm 的转换。

### 4.6 `tree3` 统计的是初始电离位置，但波形来自漂移终点

`tree3` 保存的是 `xe0/ye0/ze0`，而抽样波形依据的是 `xe1/ye1/ze1`。因此 `tree_electron_layers` 描述的是初始电离层，而解卷积出来的条电荷主要对应漂移终点在读出平面的横向扩散结果。

这并不是代码错误，但两者不能直接认为是同一层的电荷真值。当前输出同时保存它们，使用结果时必须明确：

- `tree_electron_layers`：初始电离电子的层统计；
- `ClusterDeconv.chargetier`：基于读出波形反演的 tier 电荷。

### 4.7 相关矩阵统计存在数值稳定性问题

相关系数使用：

```cpp
corr = cov / (sqrt(var1) * sqrt(var2));
```

但代码没有检查方差是否为零，也没有在所有 tier 对角线上都使用一致的有效样本集合。若某层残差恒定或有效事件不足，结果可能是 NaN 或不可靠的相关系数。

另外，`res_arr` 中用 `infinity` 表示无数据，虽然主循环跳过了 infinity，但不同 tier 对之间的有效事件数可能不同，需要在说明中报告每个矩阵元素的有效样本数。

## 5. 目前可以确认的正确部分

- 模拟保存了 `tree`、`tree2`、`tree3`，并且三棵树在每个事件中一起 `Fill()`，事件结构是对应的。
- 分析读取的是 `waveforms_conv` 分支，而不是未卷积的 `waveforms`，这与使用卷积模板进行解卷积的意图一致。
- 模拟和分析都采用 5 个 tier，整体层数参数是一致的。
- 模拟真值斜率使用 `x/y = k*z + b` 形式保存，形式上适合与 microTPC 直线拟合结果比较。
- 分析在每个事件开始清空临时 vector，避免了条或 cluster 跨事件累积。

## 6. 建议的修正优先级

### 必须先修

1. 删除 `tree3` 坐标统计和 2D 图中的错误 `*10`，或明确证明模拟坐标不是 mm；
2. 修正 `n_sig_samples` 的抽样条件，使其确实生成 15 个点；
3. 统一模拟卷积和模板卷积的边界定义；
4. 统一真值坐标、条号坐标和残差公式。

### 随后修

5. 将 `1000`、`4200`、`0.4`、`0.2` 等参数集中到配置，并写入 ROOT 文件元数据；
6. 增加输入检查：波形长度、模板长度、矩阵列数、x/y 符号和单位；
7. 对相关矩阵增加有效事件数和零方差保护；
8. 增加一个无噪声、单轨迹、已知斜率的闭环测试，要求重建结果在预期误差内回到真值。

## 7. 总结

当前模拟程序的总体流程是合理的：生成轨迹和初级电子、漂移、从抽样库累加波形、卷积并保存真值，分析程序也确实在做模板解卷积和 microTPC 重建。

但是，当前版本还不能认为“模拟真值与分析结果严格对应”。其中最严重的是 `tree3` 坐标乘以 10、14/15 个采样点不一致，以及真值残差中混用多个未统一定义的坐标换算。修正这些问题后，才能可靠评估解卷积和 microTPC 的位置/角度分辨率。
